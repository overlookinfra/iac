27th March 2020: IAC Team Status Update

Mar 27, 2020 • daianamezdrea

The IAC team has been working on a lot of interesting things over the past week. Here’s a brief overview of what we’ve been up to:

Customer Escalations

By popular request, the team started work on the Palo Alto Networks module to add support for virtual systems (check out the PR for more details).

DSC Rework

We’ve started work on re-architecting and revitalizing our DSC builder (check out IAC-41 for some details!) based on prototype work by James Pogran. In short, this project is aiming to provide a PowerShell module for Puppetizing DSC resources, complete with intellisense, help, and guardrails. A bit further out and we’ll have a new namespace on the forge - dsc - where we’ll publish these automatically generated modules as 1:1 matches for the PowerShell modules with DSC resources on the PowerShellGallery! We’re working hard right now to pull the prototype across the last couple of miles to get an MVP for testing - if you’re interested in beta testing this project with us, please reach out on slack in #windows.

Litmus Progress

On Litmus, the team had a productive week. We managed to fix some bugs that you can check out here and here. Also, now we are able to capture more information for honeycomb. Part of that is due to the new rspec_honeycomb_formatter that let’s you post all your examples and results to honeycomb.

The last few of modules where the acceptance tests need converting to Litmus are now in progress:

  • puppetlabs/websphere_application_server

  • puppetlabs/satellite_pe_tools

  • puppetlabs-reboot

  • puppetlabs-ibm_installation_manager

If you’re interested in converting a module to Litmus, feel free to get in touch with us during Office Hours or use the documentation.

Final litmus note: we did the work already a few weeks ago, but now closed the ticket: we’re now using the litmusimage docker images in all our testing. They are regularly autogenerated and accept community contributions.
